# Data Type Availability — 12 CGM Datasets (Figure 1 Style)
## Glucose-ML-Project · 실제 분석에 사용된 데이터셋 기준

**논문 Figure 1 분류 체계 (Prioleau et al., 2025) 적용**

각 데이터 카테고리:
- **G** — Continuous Glucose Monitor (연속혈당측정기)
- **I** — Insulin Delivery System (인슐린 전달 시스템)
- **A** — Activity Tracker (활동 추적기)
- **S** — Self-Report / Mobile App (자가보고 / 앱 로그)
- **Q** — Questionnaire / Survey (설문 / 인구통계)
- **M** — Medical Record / Clinical Measurement (의무기록 / 임상검사)

범례: ✅ 사용됨 (harmonized 데이터에 포함, 모델 학습에 투입) · 🔶 원본 존재하나 미포함 · ❌ 원본에 없음

## 세부 주석

> ¹ **AIDET1D — InsInjectionFreqInt / InsRoute:** 인슐린 주사 빈도(정수)와 주입 경로(MDI vs Pump). 개별 투여 타임스탬프가 아닌 환자별 치료 방식 정보이므로 I와 Q의 경계에 해당. 본 분석에서는 파생 변수 생성에 미활용 (컬럼 존재하나 0으로 채워짐).
>
> ² **IOBP2 — MealDoseAnnounceAmt:** 식사 전 볼루스 선언 용량. 인슐린 펌프 시스템의 Meal Dose Announcement — I 카테고리(탄수화물 기반 인슐린 계산 결과물).
>
> ³ **CGMND — Sleep log:** Wake_Time, Nap_SleepTime, Nap_WakeTime 3개 컬럼으로 수면-각성 패턴 기록. S(자가보고 수면 일지) 성격이지만 활동추적기 기반일 수 있음(원본 문서 불명확).
>
> ⁴ **Park\_2025 — Foods/food 컬럼:** 음식명 텍스트(한국어/중국어) 및 탄수화물 섭취 기록 포함. 텍스트 형태라 수치변수로 직접 투입 불가 → 본 분석에서 미사용. 구조화된 탄수화물값이 별도로 있을 가능성 있음.
>
> ⁵ **CGMacros — Image path:** 식사 사진 파일 경로 메타데이터. 실제 이미지파일(.jpg 수천 장)은 원본에 존재하나 .gitignore로 버전관리 제외. 본 분석에서는 수치 변수만 활용.

## 분석 Coverage 요약

| 카테고리 | 포함 데이터셋 수 | 관련 데이터셋 |
|---|:---:|---|
| **G** — CGM 혈당 | **12/12** | 전체 (100%) |
| **I** — 인슐린 | **6/12** | AIDET1D¹, Bris-T1D, HUPA-UCM, IOBP2, PEDAP, UCHTT1DM |
| **A** — 활동 추적 | **4/12** | Bris-T1D, CGMacros×2, HUPA-UCM |
| **S** — 자가보고 (식이/수면) | **6/12** | BIGIDEAs, CGMacros×2, CGMND, GLAM, Park\_2025 |
| **Q** — 설문/인구통계 | **1*/12** | AIDET1D만 harmonized 포함, 나머지는 원본에만 존재 |
| **M** — 임상검사 | **0/12** | 전체 미포함 (원본에는 일부 존재) |

> **핵심 한계:** Q(인구통계)와 M(임상검사) 카테고리는 대부분 원본 데이터셋에 존재하지만, 본 프로젝트의 harmonized 시계열 파일에는 포함되지 않음. Tier 4 이후 개인화 모델링 시 환자별 Q/M 변수를 Static Feature로 추가 투입하는 확장 가능성이 있음.
