## 1. 연구 배경과 질문

### 1.1 문제 정의

연속혈당측정(CGM) 기반 혈당 예측 모델은 **학습에 사용된 데이터셋에서는 잘 작동하지만, 다른 병원·국가·환자군의 데이터에 적용하면 성능이 급격히 하락**하는 것으로 알려져 있다. 이를 "**일반화 실패(generalizability failure)**"라 한다.

이 연구는 다음 질문에 답한다:

> **Q1.** 데이터셋이 바뀌면 성능이 얼마나 떨어지는가? (정량화)  
> **Q2.** 떨어진 성능을 전이학습(transfer learning)으로 회복할 수 있는가?  
> **Q3.** 어떤 전이 전략이 가장 효과적인가?

### 1.2 비유로 이해하기

> 서울대병원에서 수천 명의 데이터로 훈련된 혈당 예측 AI가 있다고 하자.  
> 이 AI를 **세브란스병원**에 가져가면 정확도가 크게 떨어진다 — 환자 구성, 센서 종류, 식습관이 다르기 때문이다.  
> **전이학습**은 "서울대 AI + 세브란스 데이터 소량"으로 빠르게 적응시키는 기술이다.  
> 이 보고서는 **적응 없이 그대로 쓰면 얼마나 나쁜지**, 그리고 **어떤 적응 방법이 가장 효과적인지**를 12개 데이터셋 간 144쌍의 실험으로 검증한다.

### 1.3 모델이 풀고 있는 문제: "30분 후 혈당은 몇일까?"

이 연구에서 LightGBM이 풀고 있는 문제는 다음과 같다:

```
 ┌──────── 입력 (Input) ────────┐          ┌── 출력 (Output) ──┐
 │                              │          │                   │
 │  지금부터 과거 30분간의       │  ──→     │  30분 후의         │
 │  혈당값 6개                  │  모델    │  혈당값 1개        │
 │  (5분 간격 측정)             │          │  (mg/dL)          │
 │                              │          │                   │
 │  + 그로부터 유도한            │          │  예) "142 mg/dL"  │
 │    변화율, 추세 등 14개 지표  │          │                   │
 └──────────────────────────────┘          └───────────────────┘
```

**구체적 예시:**

```
[환자 A, 오전 8:00 시점]

  입력 (과거 30분):
  ┌────────────────────────────────────────────────────┐
  │  7:30   7:35   7:40   7:45   7:50   7:55          │
  │  120 →  125 →  132 →  140 →  148 →  155 mg/dL    │  ← 아침식사 후 상승 중
  └────────────────────────────────────────────────────┘
  + Velocity = +7 mg/dL/5min (상승세)
  + TIR = 100% (6개 모두 70~180 범위)
  + tod_sin/cos = 아침 8시 인코딩
  ...등 총 20개 숫자

  모델 예측: ŷ = 172 mg/dL  (8:30의 혈당)
  실제 측정: y = 168 mg/dL  (8:30의 실제 CGM 값)
  오차: |172 - 168| = 4 mg/dL
```

> 이런 예측을 **환자 한 명당 수천~수만 번** 반복하고, 모든 환자에 대해 평균 오차를 측정한다.

### 1.4 평가 지표: "잘 맞추는 것"의 정의

모델 성능을 **여러 관점**에서 측정한다:

#### Regression 지표 (혈당값 자체의 정확도)

| 지표 | 의미 | 좋은 값 | 예시 |
|:---|:---|:---:|:---|
| **RMSE** | 평균 오차 크기 (mg/dL) | 낮을수록 ↓ | RMSE=15 → "평균적으로 15 mg/dL 틀림" |
| **MAE** | 절대 오차 평균 (mg/dL) | 낮을수록 ↓ | MAE=10 → "보통 10 mg/dL 정도 빗나감" |
| **MAPE** | 오차 비율 (%) | 낮을수록 ↓ | MAPE=8% → "실제값의 8% 정도 틀림" |

#### Post-hoc Classification 지표 (예측값을 범주로 변환 후 평가)

예측된 혈당값(mg/dL)을 임상적으로 의미있는 **범주(category)**로 변환한 뒤, 범주 단위의 일치도를 측정:

| 지표 | 변환 방법 | 의미 | 좋은 값 |
|:---|:---|:---|:---:|
| **Kappa_Range** | <70 → 저혈당, 70~180 → 정상, >180 → 고혈당 | "저혈당/정상/고혈당을 맞추는가?" | 1에 가까울수록 ↑ |
| **Kappa_Trend** | Δ<-30 → 급락, -30~-15 → 하락, ±15 → 유지, 15~30 → 상승, >30 → 급등 | "상승/하락/유지 방향을 맞추는가?" | 1에 가까울수록 ↑ |
| **Hypo_Sens** | 실제 <70인 순간 중, 모델도 <70으로 예측한 비율 | "저혈당을 놓치지 않는가?" | 1에 가까울수록 ↑ |

> **왜 Regression과 Classification을 둘 다 보는가?**
> - RMSE가 낮아도, **저혈당(<70)을 정상(100)으로 예측**하면 임상적으로 위험
> - 반대로 200과 210의 차이(RMSE 기여 10)보다, 68과 75의 차이(RMSE 기여 7)가 임상적으로 더 중요
> - Kappa_Range는 **"이 환자가 지금 위험한 구간에 있는지"**를 맞추는 능력을 직접 측정

### 1.5 이 연구에서 "성능이 떨어진다"의 의미

보고서에서 "RMSE가 17.63 → 24.23으로 악화"라고 말할 때, 이는:

> "자기 병원 데이터로 학습하면 평균 **17.63 mg/dL** 오차로 30분 후 혈당을 맞추는데,  
> 다른 병원에서 학습한 모델을 가져오면 오차가 **24.23 mg/dL**로 커진다.  
> 이 **6.6 mg/dL의 추가 오차**가 저혈당 경보의 정확도를 떨어뜨리고(Kappa 0.60→0.52),  
> 혈당 변화 방향 예측을 거의 무용하게 만든다(Kappa 0.40→0.21)."

### 2.2 공통 피처 (Global Features — 20차원)

모든 데이터셋에서 **동일하게 추출 가능한** 피처만 사용하여 공정한 비교를 보장했다:

```
[Lookback 6개]  glucose_t-5, glucose_t-4, ..., glucose_t-0  (과거 30분의 5분 간격 혈당)
[운동학 3개]    Velocity(변화율), Acceleration(가속도), Jerk(가가속도)
[통계 4개]      Window_Mean, Window_Std, Window_AUC, SD1(Poincaré 단기변동)
[임상 5개]      TIR(정상범위비율), TAR(고혈당비율), TBR(저혈당비율), LBGI, HBGI
[시간 2개]      tod_sin, tod_cos (하루 중 시각의 순환 인코딩)
                                                            합계: 20차원
```

### 2.3 데이터 분할

각 데이터셋 내에서 **환자 단위로** 70% Train / 15% Validation / 15% Test로 분할했다. 같은 환자의 데이터가 train과 test에 모두 포함되는 것(data leakage)을 방지한다.

## 3. 7가지 전이 방법론 해설

### 개념 정리: "전이학습"이란?

> 전이학습은 **이미 학습된 모델의 지식을 새로운 상황에 재활용하는 기술**이다.
> 
> - "학습": 데이터를 보고 패턴을 파악하는 과정 (LightGBM은 decision tree 300개를 순차적으로 만듦)
> - "전이": 기존에 만든 300개 tree를 새로운 데이터에서 그대로 사용하거나, 위에 tree를 추가
> - "파인튜닝": 기존 모델을 새 데이터에 맞게 **추가 훈련**하여 적응시키는 것

---

### ① self — Target 데이터만으로 학습 (Baseline)

```
  Dataset M (target)
  ┌──────────────┐
  │ Train 70%    │ ──→ LightGBM 학습 (tree 300개) ──→ Test 15%에서 평가
  │ Val 15%      │     (lr=0.1, 300 rounds)
  └──────────────┘
```

> **설명:** 가장 직관적인 방법. 예측하고 싶은 데이터셋 M의 train set으로만 모델을 학습.  
> **의미:** 다른 모든 방법의 **기준선(baseline)**. 이보다 나쁘면 전이가 해가 되는 것.

---

### ② asis_best — 가장 유사한 Source를 그대로 적용 (Zero-Shot)

```
  Dataset A (가장 유사한 source)           Dataset M (target)
  ┌──────────────┐                        ┌──────────────┐
  │ Train        │ ──→ LightGBM 학습 ──┐  │              │
  └──────────────┘                      └──→ Test에서 평가 (아무 적응 없이)
                                              ↑
                                         "A에서 배운 그대로 M에 적용"
```

> **설명:** Dataset M과 **혈당 통계가 가장 유사한** 1개 source에서 모델을 학습하고, M의 test set에 아무런 수정 없이 바로 적용.  
> **의미:** "최악의 경우" 전이. M의 데이터를 전혀 보지 않고 예측해야 할 때.

---

### ③ ft_single — 가장 유사한 Source → Fine-Tuning

```
  Dataset A (가장 유사한 source)           Dataset M (target)
  ┌──────────────┐                        ┌──────────────┐
  │ Train        │ ──→ tree 300개 학습    │ Train        │ ──→ tree 100개 추가 ──→ Test에서 평가
  └──────────────┘     (lr=0.1)           └──────────────┘     (lr=0.05)
                       ↓ 저장                                   ↑ 불러오기
                    model_A.txt ─────────────────────────────→ init_model
```

> **설명:** Source A에서 학습한 모델(tree 300개)을 저장 → M의 train set으로 **추가 학습**(tree 100개를 더 쌓음).  
> **핵심 메커니즘:**
> - LightGBM의 `init_model` 파라미터 사용
> - 기존 tree 300개는 **건드리지 않고 보존** (A에서 배운 패턴 유지)
> - 새 tree 100개가 **A와 M의 차이(잔차)를 보정** 
> - 학습률을 0.05로 낮춰 과격한 변경 방지  
> **비유:** "서울대 AI를 세브란스 데이터로 보충학습시키는 것"

---

### ④ nalm_uniform — 11개 Source의 균등 앙상블

```
  Dataset 1 ──→ model_1.predict(M_test) ─┐
  Dataset 2 ──→ model_2.predict(M_test) ─┼──→ 11개 예측의 단순 평균 = 최종 예측
  ...                                    │
  Dataset 11──→ model_11.predict(M_test) ┘
```

> **설명:** M을 제외한 11개 데이터셋 각각에서 모델을 학습 → M의 test에서 11개 예측을 만든 후 **동일한 가중치(1/11)**로 평균.  
> **의미:** "여러 전문가의 의견을 동등하게 참고하는 것"

---

### ⑤ nalm_simple — 11개 Source의 유사도 가중 앙상블

```
  Similarity Index: s₁=0.95, s₂=0.48, ..., s₁₁=0.72
                     (M과의 유사도, 높을수록 유사)

  Dataset 1 ──→ model_1.predict(M_test) × s₁ ─┐
  Dataset 2 ──→ model_2.predict(M_test) × s₂ ─┼──→ 가중 평균 = 최종 예측
  ...                                          │
  Dataset 11──→ model_11.predict(M_test) × s₁₁┘
```

> **설명:** 균등 앙상블(④)과 동일하지만, 각 source의 기여도를 **M과의 유사도에 비례**하여 결정.  
> **유사도 계산:** 혈당 평균, 표준편차, TIR, TAR, TBR 등 8개 통계의 Manhattan distance를 exponential 변환  
> **의미:** "M과 비슷한 데이터셋의 의견을 더 많이 반영"  
> **참고논문:** Nalmpatian et al. (2025)의 Country Similarity Index를 Dataset 수준으로 변환

---

### ⑥ nalm_pooled_ft — 11개 풀링 → Global Model → Fine-Tuning ⭐

```
  ┌─ Dataset 1 ─┐
  │ Dataset 2   │
  │ ...         │──→ 전부 합쳐서 1개로 ──→ Global Model (tree 300개)
  │ Dataset 11  │    (~35M 윈도우)         (모든 데이터셋의 패턴 포착)
  └─────────────┘                               │
                                                │ init_model로 전달
                                                ↓
                    Dataset M의 Train ──→ 추가 학습 (tree 100개, lr=0.05) ──→ Test에서 평가
```

> **설명:** M을 제외한 11개 데이터셋을 **물리적으로 하나로 합치고(pooling)**, 이 대규모 데이터에서 Global Model을 학습. 그 후 M의 train set으로 fine-tuning.  
> **핵심 차이:** 앙상블(④⑤)은 예측값을 합치지만, 풀링(⑥)은 **데이터 자체를 합쳐서 하나의 모델을 학습**  
> **장점:** 모든 데이터셋의 패턴을 통합적으로 파악한 후 M에 맞게 보정  
> **참고논문:** Nalmpatian et al. (2025)의 Step 1 (Global Model) + Step 2 (Specialized Model)의 결합

---

### ⑦ nalm_patient — Patient-Level Resampling + Fine-Tuning

```
  Step A: Global Model 학습 (⑥과 동일)

  Step B: Patient-Level Resampling (Nalmpatian Step 4 변형)
  ┌──────────────────────────────────────────┐
  │ 유사도 s₁=0.95 → Dataset 1에서 많이 선택  │
  │ 유사도 s₂=0.48 → Dataset 2에서 적게 선택  │──→ 합성 데이터셋 구성
  │ ...                                      │    (유사한 source의 데이터를 더 많이 포함)
  │ 유사도 s₁₁=0.72→ Dataset 11에서 중간 선택  │
  └──────────────────────────────────────────┘
                                  ↓
               Global Model + 합성 데이터로 중간 학습 (tree 100개)
                                  ↓
               + Dataset M의 Train으로 최종 fine-tuning (tree 50개)
                                  ↓
               Test에서 평가
```

> **설명:** 3단계 학습.
> 1. Global Model: 11개 전체 풀링 학습 (공통 패턴)
> 2. 유사도 비례 리샘플링: M과 유사한 source의 데이터를 더 많이 선택하여 중간 학습 (M에 가까운 패턴 강화)
> 3. M으로 fine-tuning (최종 적응)
>
> **원본 논문과의 차이:** Nalmpatian et al.은 **행 단위 랜덤 리샘플링 + 노이즈 주입**을 했지만, 시계열 CGM 데이터에서는 시간 순서가 파괴되므로 **환자(윈도우 블록) 단위 선택**으로 대체.  
> **비유:** "서울대+연세대+고려대+... 전체 데이터로 기본을 배우고 → 세브란스와 비슷한 병원의 데이터를 집중 학습 → 세브란스 소량 데이터로 마무리"

### 6.4 주요 발견

#### 발견 1: nalm_pooled_ft가 self보다 우수

> **다른 11개 데이터셋의 데이터를 풀어서 Global Model을 만들고, target으로 fine-tuning한 것이, target 데이터만으로 학습한 것보다 평균 0.44 mg/dL 더 정확하다.**

이는 **"데이터가 많을수록 좋다"**는 직관을 정량적으로 확인한 결과. 데이터셋 간 코호트가 달라도, 혈당 dynamics의 공통 패턴(식후 상승, 인슐린 효과, 야간 하락 등)은 전이 가능하다.

특히 소규모 데이터셋에서 효과가 극적:
- **CGMacros_Libre:** self(4.92) → nalm_pooled_ft(**1.57**), **68% 개선**
- **BIGIDEAs:** self(13.90) → nalm_patient(**13.04**), 6% 개선
- **UCHTT1DM:** self(15.13) → nalm_pooled_ft(**14.65**), 3% 개선

#### 발견 2: 단순 앙상블은 self보다 나쁘다

nalm_simple(20.73)과 nalm_uniform(21.18) 모두 self(17.63)보다 나쁘다. **여러 모델의 예측 평균**은 개별 모델들이 서로 다른 오차를 가질 때만 효과적인데, 코호트가 극단적으로 다른 경우(ND→T1DM) 예측 자체가 편향(bias)되므로 평균이 도움이 안 된다.

> 하지만 유사도 가중 앙상블(20.73)이 균등 앙상블(21.18)보다 **0.45 mg/dL 낫다** — 유사도의 효과는 존재.

#### 발견 3: 풀링 > 앙상블

| 전략 | RMSE | 차이 |
|:---|:---:|:---:|
| ④ nalm_uniform (11개 모델 평균) | 21.18 | — |
| ⑤ nalm_simple (유사도 가중 평균) | 20.73 | -0.45 |
| nalm_pooled (11개 풀링, 적응 없음) | 19.10 | **-2.08** |

**데이터를 합쳐서 1개 모델을 만드는 것**이 각각 모델을 만들어 평균하는 것보다 2 mg/dL 이상 좋다. 이는 풀링이 데이터셋 간 **공유 패턴**을 더 효과적으로 학습하기 때문.

#### 발견 4: Trend prediction에서 전이학습의 효과가 가장 크다

| 방법 | Kappa_Trend | vs self |
|:---|:---:|:---:|
| self | 0.396 | — |
| **nalm_pooled_ft** | **0.424** | **+0.028** |
| nalm_patient | 0.410 | +0.014 |

**혈당 변화 방향(Trend) 예측은 cross-dataset 적용 시 가장 취약한 지표였으나(Phase 2에서 0.21까지 하락), 전이학습(특히 nalm_pooled_ft)이 self보다도 더 좋은 성능을 달성.** 다양한 데이터셋의 패턴을 학습함으로써 trend prediction의 일반성이 향상된 것.

#### 발견 5: 대규모 데이터셋에서는 전이학습 이점이 작다

| 데이터셋 | 환자 수 | self RMSE | nalm_pooled_ft RMSE | Δ |
|:---|:---:|:---:|:---:|:---:|
| CGMacros_Libre | 30 | 4.92 | **1.57** | **-3.35** |
| UCHTT1DM | 20 | 15.13 | **14.65** | -0.48 |
| GLAM | 886 | **13.72** | 13.78 | +0.06 |
| IOBP2 | 440 | **24.59** | 24.64 | +0.05 |
| PEDAP | 103 | **29.27** | 29.30 | +0.03 |

> GLAM, IOBP2, PEDAP 같은 대규모 데이터셋은 **자체 데이터만으로도 충분**하여 전이학습의 추가 이점이 거의 없다. 반면, 소규모 데이터셋에서는 전이학습의 이점이 극적으로 나타난다.

## 8. 한계와 향후 과제

### 8.1 현재 한계

| 한계 | 설명 | 영향 |
|:---|:---|:---|
| **공통 피처만 사용** | 인슐린, 식사, 활동량 등 고유 피처 미사용 | Specialized Model의 진정한 가치 검증 불가 |
| **Q/M 정적 피처 미포함** | 나이, HbA1c, BMI 등 환자 특성 미반영 | 환자 간 이질성 포착 부족 |
| **LightGBM 단일 모델** | 딥러닝(LSTM, Transformer) 미비교 | 모델 구조에 따른 전이 효과 차이 미확인 |
| **Patient-Level Resampling 근사** | 현재 구조에서 환자 ID가 X에 없어 윈도우 비례로 근사 | 정확한 환자 단위 선택이 아님 |

### 8.2 향후 과제

1. **Q/M 정적 피처 통합:** age, sex, HbA1c, BMI, DM duration을 Global Feature로 추가 → Nalmpatian의 "demographic features" 대응
2. **Specialized Model 구현:** 각 데이터셋의 고유 피처(인슐린, 영양, 활동량)를 활용하는 Local Model 학습
3. **Drift Analysis:** 전이 오차의 원인을 코호트/센서/피처 수준으로 분해 (GLM 기반)
4. **딥러닝 비교:** LSTM, Temporal Fusion Transformer와의 전이 성능 비교
