# Tier 4 회의 안건
# CGM 혈당 예측 모델의 재현성과 일반화 가능성

> **일시:** 2026-04-15  
> **Target Journal:** npj Digital Medicine  

---

## 배경: 이 연구가 무엇인가?

### 연구 대상

**연속혈당측정기(CGM, Continuous Glucose Monitor)** 는 피부 아래에 부착하여 5~15분마다 혈당을 자동 측정하는 웨어러블 기기이다. 이 기기가 생성하는 시계열 데이터를 사용해 **30분 후 혈당을 예측**하는 머신러닝 모델을 만드는 것이 목표이다.

### 핵심 문제

현재 대부분의 혈당 예측 연구는 **하나의 데이터셋**에서 모델을 만들고 평가한다. 그러나 실제 병원에서 이 모델을 쓰려면, **다른 병원, 다른 환자군, 다른 센서**에서 수집한 데이터에서도 잘 작동해야 한다. 이것이 가능한지를 체계적으로 검증한 연구는 CGM 분야에서 거의 없다.

### 우리의 접근

**12개의 공개 CGM 데이터셋**을 모아서, 다음 두 가지를 정량적으로 분석한다:

1. **Within Variation (내부 변동):** 같은 데이터셋에서 하이퍼파라미터 설정이나 랜덤 시드를 바꾸면 결과가 얼마나 달라지는가?
2. **Between Variation (외부 변동):** 한 데이터셋에서 만든 모델을 **다른 데이터셋**에 적용하면 성능이 얼마나 떨어지는가? 전이학습(Transfer Learning)으로 이 격차를 줄일 수 있는가?

### 용어 정리

| 용어 | 설명 |
|:---|:---|
| **CGM** | 연속혈당측정기. 5~15분마다 혈당 기록 |
| **RMSE** | 예측 오차 (mg/dL). 낮을수록 좋음. 0이면 완벽한 예측 |
| **GBM** | Gradient Boosting Machine. 의학 분야에서 많이 쓰이는 머신러닝 모델 (XGBoost, LightGBM, CatBoost 등) |
| **HPO** | Hyperparameter Optimization. 모델의 학습 세팅을 자동 최적화하는 과정 |
| **LODO** | Leave-One-Dataset-Out. 12개 중 1개를 빼고 나머지 11개로 학습 → 뺀 1개로 평가. 이를 12번 반복 |
| **Tier** | 우리 프로젝트의 실험 단계. Tier 3 = 단일 데이터셋 기준선, Tier 4 = 다중 데이터셋 전이학습 |

## 0. 현재까지의 실험 결과 요약

### 0.1 데이터 준비 ✅

12개 공개 CGM 데이터셋을 통일된 형식으로 전처리했다. 데이터셋 간 규모와 특성이 매우 다르다:

| 특성 | 범위 | 시사점 |
|:---|:---|:---|
| 피험자 수 | 16 ~ 886명 | 55배 차이 |
| 총 측정포인트 | 24K ~ 26.6M | 1,000배 차이 |
| 수집 기간 | 7일 ~ 5.3년 | 280배 차이 |
| 평균 혈당 | 100 ~ 170 mg/dL | 당뇨 유형에 따라 크게 다름 |
| 센서 간격 | 5분 또는 15분 | 해상도 3배 차이 |
| 환자군 | T1DM, GDM, ND, PreD | 질환 유형 혼재 |

**특이사항:** 모든 데이터셋의 원본에는 나이, 성별, HbA1c(당화혈색소), BMI 같은 **환자 기본 정보**가 존재하지만, 현재 파이프라인에서는 이를 **사용하지 않고** 순수하게 혈당 시계열 패턴만으로 예측하고 있다. (→ 안건 1.3에서 논의)

### 0.2 Tier 3 결과: 단일 데이터셋 모델 성능 ✅

각 데이터셋에서 개별적으로 GBM 모델 3종(XGBoost, LightGBM, CatBoost)을 학습하고 평가했다.

**핵심 발견:**

| 발견 | 수치 | 의미 |
|:---|:---|:---|
| RMSE 범위 | 1.34 ~ 28.38 mg/dL | **데이터셋에 따라 성능 차이가 20배 이상** |
| HPO(4.6시간 소요) 후 개선 | **평균 0.004 mg/dL** | 사실상 **아무 효과 없음** |
| 가장 큰 성능 결정 요인 | 환자군 특성 (T1DM vs ND) | 모델이나 HPO보다 **데이터 자체**가 중요 |

> **결론: 같은 데이터에서 하이퍼파라미터를 바꿔도 성능은 거의 변하지 않는다 (Within Variation ≈ 0). 반면 데이터셋을 바꾸면 성능이 극적으로 변한다 (Between Variation >> 0). 따라서 연구 초점은 데이터셋 간 전이학습(Between Variation)에 두어야 한다.**

### 0.3 Tier 4 문헌 조사: 전이학습 방법론 ✅

보험 사망률 예측에서 GBM 기반 전이학습을 제시한 논문(Nalmpatian et al., 2025)의 방법론을 분석했다. 해당 논문은:
- **유사도 지수(Similarity Index)** 로 source와 target의 유사도를 계산하고
- 유사도에 비례하여 **합성 데이터**를 생성한 뒤
- **2단계 GBM(Global → Specialized)** 으로 예측하는 프레임워크를 제시한다.

**문제 발견:** 이 방법의 핵심인 "합성 데이터 생성"(데이터를 무작위로 섞어서 복제 + 노이즈 주입)은 **시계열 데이터인 CGM에 적용 불가능**하다. 혈당 데이터는 시간 순서가 핵심인데, 무작위 섞기는 시간 순서를 파괴하기 때문이다. 이는 CGM 벤치마크 기준 논문(Prioleau et al., 2025)이 명시적으로 금지하는 데이터 조작에 해당한다.

**대안:** "합성 데이터 생성" 대신 **환자 단위 선택(Patient-Level Transfer)** 을 제안했다. 이는 유사한 데이터셋에서 더 많은 환자의 원본 데이터를 가져오되, 데이터 자체는 한 글자도 변형하지 않는 방법이다.

## 1. 논의 안건

### 안건 1.1 — 실험 구조: Within vs Between을 어떻게 설계할 것인가?

참고문헌인 Yang et al. (2022)은 4개 영국 병원에서 COVID-19 스크리닝 모델의 일반화 가능성을 연구했다. 이 논문은 모델을 새 병원에 적용하는 3가지 방법을 비교했다:

| Yang et al.의 방법 | 설명 | 우리 프로젝트에서의 대응 |
|:---|:---|:---|
| **(1) As-is** | A병원 모델을 B병원에 **그대로** 적용 | 데이터셋 A에서 학습한 모델을 데이터셋 B에 바로 적용 |
| **(2) Threshold** | 모델은 그대로 두고 **판정 기준만** 새 병원 데이터로 재조정 | GBM 예측값의 offset(편향)만 새 데이터셋으로 보정 |
| **(3) Transfer** | 모델을 새 병원 데이터로 **미세 조정(fine-tune)** | 새 데이터셋의 환자 데이터로 모델 weight 업데이트 |

**결정이 필요한 사항:**

- [ ] **Within Variation 실험을 어떻게 구성할 것인가?**
  - (A) HPO 변동: 이미 완료 (Δ ≈ 0 확인됨)
  - (B) 랜덤 시드 변동: 같은 데이터에서 train/test 나누는 기준만 바꿔서 5~10회 반복 → **추가 실험 필요**
  - (C) 피처 구성 변동: 환자 기본 정보(나이, HbA1c 등) 포함 vs 미포함 비교

> **참고: HPO 실험 결과 상세**  
> 12개 데이터셋 × LightGBM/CatBoost 2종에 대해 Optuna로 하이퍼파라미터 최적화를 수행했다 (총 4.6시간 소요). 결과: 12개 중 5개만 소폭 개선(최대 -0.33 mg/dL), 4개는 오히려 악화, 3개는 변화 없음. **전체 평균 개선은 0.004 mg/dL**로, CGM 센서 자체의 측정 오차(±10~15 mg/dL)의 0.04%에 불과하다. 이는 GBM의 디폴트 하이퍼파라미터가 이미 거의 최적임을 의미하며, **실제 병원 환경에서 HPO를 위한 추가 컴퓨팅 자원을 투입할 필요가 없다**는 실용적 결론을 제공한다. 즉 Within Variation의 HPO 축은 이미 "변동 없음"으로 결론나 있으므로, 추가 실험이 필요하다면 (B) 랜덤 시드 축이 대상이 된다.

- [ ] **Between Variation 실험을 어떤 형태로 할 것인가?**
  - (A) LODO: 11개로 학습 → 1개로 평가 (12회 반복) — 가장 표준적
  - (B) 모든 쌍 비교: A→B, A→C, ..., L→K (12×11 = 132회) — 가장 정보량 많지만 시간 소요 큼
  - (C) Yang et al. 스타일: as-is / threshold / transfer 3단계를 모든 쌍에 적용

### 안건 1.2 — "결과의 일관성"을 어떤 지표로 측정할 것인가?

> 핵심: 모델 A가 예측한 결과와 모델 B가 예측한 결과가 **얼마나 비슷한지** 를 숫자로 표현해야 한다.

#### 후보 지표

| 지표 | 무엇을 측정하는가 | 적합한 상황 | 해석 예시 |
|:---|:---|:---|:---|
| **Pearson 상관계수** | 두 예측값 간 선형 관계 | 모델 A와 B의 환자별 예측값을 비교 | 1.0 = 완벽히 같은 패턴, 0 = 무관 |
| **ICC** | 여러 모델이 같은 환자에 대해 얼마나 일치하는가 | 12개 모델의 동일 환자 예측값 비교 | 0.9+ = 매우 높은 재현성 |
| **Cohen's Kappa** | 두 모델의 분류가 얼마나 일치하는가 | 예측 혈당을 "저혈당/정상/고혈당"으로 나눈 뒤 비교 | 0.8+ = 거의 완벽한 일치 |
| **CCC** | 정밀도와 정확도를 동시에 평가 | 예측값과 실측값의 일치도 | RMSE보다 정밀한 일치도 평가 |

**결정이 필요한 사항:**
- [ ] 주 일관성 지표를 어떤 것으로 할 것인가? (논문 메인 Figure에 사용)
- [ ] Cohen's Kappa 사용 시 혈당 범주 경계: 70/180 mg/dL (국제 기준) 사용?

### 안건 1.3 — 환자 기본 정보(Q/M 데이터)를 모델에 추가할 것인가?

현재 모델은 CGM 혈당 시계열만 사용한다. 그러나 원본 데이터에는 다음 정보도 있다:

| 변수 | 설명 | 몇 개 데이터셋에 존재? |
|:---|:---|:---:|
| 나이, 성별 | 인구통계 | 12/12 (전체) |
| HbA1c | 최근 2~3개월 평균 혈당 지표 | 7/12 |
| BMI | 체질량지수 | 7/12 |
| 당뇨 유병기간 | 진단 후 경과 년수 | 8/12 |

> **비유:** 현재 모델은 환자의 혈당 그래프만 보고 "30분 후 혈당이 150이 될 것"이라 예측하지만, "이 환자가 T1DM인지 건강한 성인인지"는 전혀 모른다. HPO로 0.004 mg/dL 개선을 추구하는 것보다, 이 정보를 알려주는 것이 더 큰 효과를 낼 수 있다.

**결정이 필요한 사항:**
- [ ] 이 피처를 이번 논문에 포함하여 실험할 것인가, 아니면 후속 연구로 분리할 것인가?
- [ ] 포함 시, 일부 데이터셋에 없는 변수(예: HbA1c가 5/12에만 존재)의 결측 처리는 어떻게?

### 안건 2.1 — 전이학습 방식 선택

"합성 데이터 생성"이 불가능하므로, 3가지 대안을 제시했다:

| 대안 | 핵심 아이디어 | 장점 | 단점 |
|:---|:---|:---|:---|
| **A. 모델 융합** | 11개 모델의 예측을 유사도 가중 평균 | 구현 쉬움 | 각 모델의 고유 피처 활용 불가 |
| **B. 가중 학습** | 11개 데이터를 유사도 비례 가중치로 합쳐서 1개 모델 학습 | LightGBM 한 줄로 구현 | 2단계 구조 포기 |
| **C. 환자 단위 선택 (권장)** | 유사한 데이터셋에서 더 많은 환자의 **원본 데이터**를 선택 | 시계열 100% 보존, 데이터 무변형 | 환자 수 적은 데이터셋에서 선택지 제한 |

**결정이 필요한 사항:**
- [ ] C만 실행할 것인가, A/B/C 모두 비교 실험할 것인가?
- [ ] Yang et al.의 3단계(as-is → threshold → transfer) 비교 구조를 1:1로 따를 것인가?

### 안건 2.2 — 데이터셋 유사도(Similarity Index)는 무엇으로 계산할 것인가?

전이학습에서 "어떤 source 데이터셋을 더 많이 참조할 것인가"를 결정하는 유사도 점수가 필요하다. 후보 지표:

| 범주 | 지표 | 직관적 설명 |
|:---|:---|:---|
| 혈당 통계 | 평균 혈당, 변동성(SD) | "평균 혈당이 비슷한 데이터셋이 예측에 유리한가?" |
| 임상 범위 | 정상 범위 비율(TIR), 고혈당 비율(TAR) | "고혈당 패턴이 비슷한 데이터셋끼리 전이가 잘 되는가?" |
| 코호트 | 환자군 유형 (T1DM/ND/GDM) | "같은 당뇨 유형이 가장 중요한 유사도 요인인가?" |
| 데이터 구조 | 센서 간격, 수집 기간 | "센서가 같으면 전이가 더 잘 되는가?" |

**결정이 필요한 사항:**
- [ ] 어떤 지표를 포함할 것인가? (전부 vs 선별)
- [ ] 유사도 행렬(12×12 히트맵)을 논문의 Figure로 제시할 것인가?

| # | 기여 후보 | 유형 | 현재 상태 |

### 안건 3.2 — 연구 결과는 어떤 형태로 보여줄 것인가?

| 산출물 | 내용 | 시각화 예시 |
|:---|:---|:---|
| **재현성 비교** | Within(HPO/시드 변동) vs Between(LODO 변동) 크기 비교 | 박스플롯: Within은 납작하고 Between은 넓음 |
| **전이 성능 행렬** | 12×12 히트맵 — "A에서 학습→B에서 평가" 모든 쌍의 RMSE | 히트맵: 대각선(자기 자신)이 가장 밝음 |
| **유사도 vs 전이 성능** | 유사도가 높은 쌍일수록 전이 성능이 좋은가? | 산점도: X=유사도, Y=ΔRMSE |
| **방법론 비교** | as-is vs threshold vs transfer 3단계 성능 비교 | 막대 그래프: Yang et al. Fig.2 스타일 |
