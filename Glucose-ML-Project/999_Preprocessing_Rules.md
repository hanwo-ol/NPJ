# Data Preprocessing & Cleaning Rules (Based on 2507.14077v1)

이 문서는 제공된 논문(`2507.14077v1.pdf`, Glucose-ML)의 `5.2 Data Cleaning` 및 관련 챕터들에서 명시된 데이터 전처리 방법론을 조사하여 수립한 전처리 기준입니다. 향후 파이프라인 개발 시 이 기준을 준수해야 합니다.

## 1. 값의 유효성 검증 (Range Filtering)
- **비수치 데이터 제거:** "LOW", "HIGH" 등 비수치(Non-numeric) 텍스트 값을 제거합니다.
- **표준 측정 범위 필터링:** 40 mg/dL 미만 또는 400 mg/dL 초과의 수치를 이상치(Outliers)로 간주하여 데이터셋에서 제외합니다.

## 2. 비현실적 변화율 제거 (Rate of Change Filtering)
- **센서 오류 필터링:** 생리학적으로 불가능한 수준의 혈당 변화 기록을 제거합니다 (DeSalvo and Buckingham 논문 기준).
  - 두 측정값 사이의 간격이 **30초 미만**인 경우.
  - 두 측정값 사이의 혈당 변화율(Rate of Change)이 **20 mg/dL/min 을 초과**하는 경우 (센서 노이즈로 간주).

## 3. 결측치(Missing Data) 정의 및 엄격한 배제 (No Interpolation)
- **결측 구간 기준:** 특정 CGM 기기의 샘플링 주기의 **3배(3x)** 이상 간격이 발생하면 결측 기간(Period of missing data)으로 처리합니다.
  - 5분 주기 센서 $\rightarrow$ 15분 이상 공백 시 결측
  - 15분 주기 센서 $\rightarrow$ 45분 이상 공백 시 결측
- **보간(Interpolation) 금지:** 선형 보간(Linear Interpolation) 등 수학적 보간 기법의 사용을 배제합니다. 예측 오류를 방지하기 위해 결측 기간이 포함된 데이터는 분석에서 제외합니다. (논문 5.3장 참고)
- **시퀀스 제외:** 결측 기간이 포함된 윈도우는 데이터셋 구성에서 제외합니다.

## 4. 데이터 적합성 기준 (Data Sufficiency Threshold)
- **일일 타당성 검사:** 24시간 동안 기록된 CGM 샘플의 양이 기대 샘플 수의 **70% 미만**일 경우, 해당 날짜의 데이터는 TIR, TAR, TBR 등 임상 지표 산출 시에만 제외한다.
- **학습 윈도우 추출 단계에는 이 기준을 적용하지 않는다.** 70% 미만 날짜라도 윈도우 내부에 NaN 또는 3x 갭이 없으면 학습에 사용한다.

## 5. 시계열 분리 원칙 (Temporal Integrity Rule)
- 시계열 데이터에는 `shuffle=True` 방식의 무작위 Train/Test 분리를 적용하지 않는다. 인접 윈도우 간 샘플 중복으로 테스트 성능이 과대 추정된다.
- 분리 스키마: Subject(환자) 단위 3-way 분리를 사용한다.
  - **Train 70%**: 모델 파라미터 학습에 사용한다.
  - **Val 15%**: Early Stopping 및 하이퍼파라미터 선택에 사용한다. 모델 학습 중 검증용으로만 사용한다.
  - **Test 15%**: 최종 성능 보고에 1회만 사용한다. 모델 선택 또는 파라미터 조정에 사용하지 않는다.
- Subject 배정은 고정 Seed(예: 42)로 무작위로 결정하며, Subject 내부 시계열 순서는 보존한다.
- `sklearn.train_test_split`의 기본 옵션(`shuffle=True`)은 시계열 실험에서 사용을 금지한다.

## 6. 스텝 수 기반 태스크 정의 원칙 (Step-Count Task Definition Rule)
- 예측 지평선과 룩백 구간은 샘플 스텝 수로 Config에 정의한다. 시간 단위(분)로 정의하면 샘플링 주기가 다른 데이터셋 간 피처 차원이 불일치한다.
- 스텝 수를 고정하면 샘플링 주기에 따라 물리적 예측 시간이 달라진다. 이를 명시적으로 문서화해야 한다.
  - 예: `LOOKBACK_STEPS = 3`, `PREDICTION_STEPS = 3` 고정 시, 5분 주기 센서는 15분 뒤 예측, 15분 주기 센서는 45분 뒤 예측.
- 이미 물리 시간 단위로 정의된 기존 코드는 스텝 수 기반으로 수정한다.
- `LOOKBACK_STEPS`와 `PREDICTION_STEPS`의 단일 진실 원본은 프로젝트 루트의 `global_config.py`이다. 각 Tier Config는 이 값을 상속하며, 재정의하지 않는다.

## 7. 전체 윈도우 유효성 검사 원칙 (Full-Window Validity Rule)
- 슬라이딩 윈도우의 유효성 검사(NaN 여부, 3x 갭 여부)는 룩백 구간과 타겟 지점 사이의 **전체 구간(t-(n_back-1) ~ t+n_fwd)**을 대상으로 수행한다.
- 룩백과 타겟 끝점만 검사하고 중간 구간을 누락하는 부분 검사를 금지한다.

## 8. 태스크 정의 (Task Definition)
- 이 프로젝트의 기본 예측 태스크는 **3 스텝 예측**이다: `LOOKBACK_STEPS = 3`, `PREDICTION_STEPS = 3`.
- 물리적 예측 시간은 그룹마다 다르다. 결과 보고 시 그룹별 실제 예측 시간을 명시한다.

## 9. 데이터셋 제외 기준 (Dataset Exclusion Criteria)
다음 조건 중 하나에 해당하는 데이터셋은 파이프라인에서 제외하고, Config의 `EXCLUDED_DATASETS`에 등록한다:
- `timestamp` 컬럼이 절대 시각이 아닌 경우. 예: 식사 기준 상대 시간(분), 인덱스 번호 등.
- `extracted-glucose-files` 폴더 자체가 이미 선형 보간된 데이터이고, 원본 비보간 데이터를 확보할 수 없는 경우.
- 현재 제외 목록: `Park_2025` (timestamp가 식사 기준 상대 시간이므로 연속 CGM 예측 불가)
